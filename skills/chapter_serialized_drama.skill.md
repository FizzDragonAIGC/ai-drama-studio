# 連載劇章節拆分法
> 讓觀眾追著看的劇集設計

## 🎯 核心理念
**每集都是一個獨立的情感體驗，同時又是更大故事的一部分。**

---

## 📋 連載劇的章節結構

### 單集結構（以10分鐘AI番劇為例）

```
[00:00-00:30] 冷開場 (Teaser)
    └── 抓住觀眾，回顧上集或新懸念

[00:30-02:00] 設置 (Setup)
    └── 本集的主要衝突/目標

[02:00-05:00] 發展 (Development)
    └── 嘗試解決問題，遇到障礙

[05:00-07:00] 轉折 (Complication)
    └── 事情變得更糟或出現新信息

[07:00-09:00] 高潮 (Climax)
    └── 本集的情感高點

[09:00-10:00] 結局+鉤子 (Resolution+Hook)
    └── 部分解決 + 新懸念
```

---

## 🎬 A/B 劇情線設計

### 主線 (A Story) - 60-70%時長
- 本集的主要衝突
- 推動整體故事弧線

### 副線 (B Story) - 30-40%時長
- 配角的故事 / 愛情線 / 主題深化
- 與主線形成對照或平行

### 交織技巧
```
A1 → B1 → A2 → B2 → A3(高潮) → B3 → A4+B4(匯合)
```

---

## 📊 連載劇的節奏模型

### 「二四八」法則
- **第2集**: 第一個小高潮（確認追劇）
- **第4集**: 中前期轉折（加深投入）
- **第8集**: All Is Lost（情感最強）

### 每集情緒曲線
```
    ↗ 緊張
   /    ↘ 舒緩
  /       ↘
起 → 承 → 轉 → 合 → 鉤子
```

## 🎯 章節拆分決策樹

### 這段內容應該是...

```
這是關鍵轉折點嗎？
  ├── 是 → 獨立成一集的高潮
  └── 否 → 繼續判斷
        │
        這是新場景/新衝突嗎？
        ├── 是 → 可能是新集的開始
        └── 否 → 合併到當前集
              │
              這段是否改變了價值/情緒？
              ├── 是 → 保留，作為發展部分
              └── 否 → 考慮刪減或改寫
```

---

## 💡 確保每集有趣的檢查清單

### 結構檢查
- [ ] 有清晰的本集目標嗎？
- [ ] 有至少一個轉折嗎？
- [ ] 有情緒高點嗎？
- [ ] 結尾有鉤子嗎？

### 角色檢查
- [ ] 主角在這集有成長/變化嗎？
- [ ] 配角有功能性作用嗎？
- [ ] 有角色關係的發展嗎？

### 節奏檢查
- [ ] 有快慢交替嗎？
- [ ] 開頭能抓住觀眾嗎？
- [ ] 時長合適嗎？

## 🎬 長篇改編的章節分配

### 10萬字小說 → 10集AI番劇

| 小說位置 | 對應集數 | 重點 |
|----------|----------|------|
| 0-10% | 第1集 | 建立世界觀 |
| 10-20% | 第2集 | 觸發事件 |
| 20-30% | 第3集 | 進入新世界 |
| 30-40% | 第4集 | 嘗試與挫折 |
| 40-50% | 第5集 | 中點轉折 |
| 50-60% | 第6集 | 反撲開始 |
| 60-70% | 第7集 | 困境加深 |
| 70-80% | 第8集 | 最黑暗時刻 |
| 80-90% | 第9集 | 終極對決 |
| 90-100% | 第10集 | 解決與新平衡 |
