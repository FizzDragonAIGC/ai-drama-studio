# 📝 懸念設計技能

## 元信息
- **技能ID**: outline_cliffhanger
- **所屬Agent**: 大綱Agent
- **參考書籍**: 《故事》+ 《編劇的藝術》
- **難度**: ⭐⭐⭐⭐

## 核心理念

懸念 = 未解答的問題 × 情感投資
- 觀眾必須在乎答案
- 答案必須延遲揭曉

## 懸念層級

### 微懸念（場景級）
```
持續：30秒-3分鐘
例：「門後面是誰？」

功能：保持注意力
```

### 小懸念（段落級）
```
持續：5-15分鐘
例：「他會同意嗎？」

功能：推動觀看
```

### 中懸念（集級）
```
持續：整集
例：「他能完成任務嗎？」

功能：結構骨架
```

### 大懸念（季級）
```
持續：整季
例：「真兇是誰？」

功能：追劇動力
```

## 懸念製造技術

### 1. 延遲揭曉
```
給出問題 → 插入其他內容 → 回到問題

例：
- 角色A打開門...
- 切到角色B的場景
- 回到A：「原來是你」
```

### 2. 限制信息
```
觀眾知道的 < 角色知道的
= 好奇型懸念

觀眾知道的 > 角色知道的
= 焦慮型懸念（希區柯克炸彈理論）
```

### 3. 錯誤方向
```
暗示答案A → 觀眾相信 → 揭露是B

紅鯡魚技巧
```

### 4. 疊加懸念
```
未解決懸念1 + 新懸念2 + 新懸念3
= 複雜但不混亂

規則：同時不超過3個大懸念
```

## 懸念payoff規則

### payoff時機
```
微懸念：必須在同場景解決
小懸念：必須在同集解決
中懸念：可以延續1-2集
大懸念：季終前必須解決（至少部分）
```

### payoff質量
```
滿意型payoff：
- 既意外又合理
- 有足夠鋪墊
- 情感上有分量

失望型payoff（避免）：
- 太容易猜到
- 沒有邏輯
- 虎頭蛇尾
```

## 懸念地圖模板

```
季度懸念追蹤：

大懸念：誰是內鬼？
├── 集1：引入問題
├── 集3：懷疑角色A
├── 集5：A被洗白，懷疑B
├── 集8：B似乎是內鬼
├── 集10：反轉，是C
└── 集12：揭曉，原來是D（A+C都是煙幕）

中懸念：主角的過去
├── 集2：暗示有秘密
├── 集6：揭露部分
└── 集9：完全揭曉
```
