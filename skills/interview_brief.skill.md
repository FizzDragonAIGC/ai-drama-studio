# 🎤 創意簡報合成技能

## 元信息
- **技能ID**: interview_brief
- **所屬Agent**: 訪談Agent
- **參考書籍**: 《廣告創意的突破》John Hegarty
- **難度**: ⭐⭐⭐

## 核心理念

創意簡報 = 所有訪談的結晶
- 一份文件統一所有人的理解
- 是創作的「憲法」，遇到爭議就回來看

## 創意簡報結構

### 1. 項目概述
```
項目名稱：
項目類型：（短劇/番劇/電影）
預計長度：（集數 × 每集時長）
目標平台：
預計上線時間：
```

### 2. 核心定義
```
Logline（一句話概括）：
[主角] 必須 [行動] 否則 [後果]

高概念（Hook）：
[類型A] 遇上 [類型B]
或：「如果...會怎樣？」

核心情緒：
看完觀眾的主要感受是 [情緒]
```

### 3. 主題與價值
```
主題陳述：
關於 [話題]，我們想說 [觀點]

核心價值觀：
- 信念1
- 信念2

觀眾帶走什麼：
看完這部作品，觀眾會 [思考/感受/行動]
```

### 4. 目標受眾
```
核心受眾：
[年齡]、[特徵]、[興趣]

受眾痛點：
他們 [現實困境]，渴望 [理想狀態]

為什麼這個故事跟他們有關：
[共鳴點]
```

### 5. 視覺風格
```
整體調性：
[形容詞1]、[形容詞2]、[形容詞3]

色彩：
主色調：
輔色調：

參考作品：
- 視覺：[作品]
- 敘事：[作品]
- 情緒：[作品]

絕對避免：
- [反參考]
```

### 6. 故事核心
```
主角：
[名字] - [一句話描述] - [核心衝突]

對手：
[名字] - [為什麼是威脅] - [與主角關係]

主線：
開始：主角是 [狀態]
觸發：[事件] 打破日常
衝突：主角必須面對 [障礙]
高潮：主角選擇 [行動]
結局：主角變成 [狀態]

核心問題：
故事要回答「[問題]？」
```

### 7. 成功指標
```
質量標準：
- 觀眾說「[理想評價]」
- 被比作「[目標作品]」

數據指標：
- 播放量目標
- 完播率目標
- 討論熱度

長期目標：
- 是否有續作計劃
- 是否發展為IP
```

### 8. 限制條件
```
預算：
技術限制：
時間限制：
其他約束：
```

## 簡報檢查清單

```
□ Logline 能在10秒內說清楚？
□ 主題清晰且有觀點？
□ 目標受眾明確？
□ 視覺參考至少3個？
□ 主角有清晰的弧光？
□ 衝突有足夠賭注？
□ 與市場現有作品有區隔？
□ 所有關鍵決策者都同意？
```

## 簡報會議流程

```
1. 展示簡報（10分鐘）
   - 項目發起人朗讀核心部分

2. 提問環節（15分鐘）
   - 每個部門提出疑問

3. 對齊環節（10分鐘）
   - 確認關鍵理解一致

4. 簽字確認（5分鐘）
   - 各部門負責人簽署

5. 存檔
   - 上傳到項目文檔庫
   - 設為只讀（修改需走流程）
```

## 簡報修改原則

```
重大修改（需要新會議）：
- 主題改變
- 核心受眾改變
- 視覺風格大改

小修改（可以郵件確認）：
- 參考作品增減
- 措辭調整
- 細節補充

禁止修改（除非重開項目）：
- Logline
- 高概念
- 核心情緒
```
