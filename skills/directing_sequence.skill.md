## 📐 什么是Sequence（段落/序列）

**定义**：由多个场景组成的叙事单元，有自己的小高潮

### 影片结构层级
```
┌───────────────────────────────────────┐
│               整部影片                │
├───────────────────────────────────────┤
│   第一幕    │    第二幕    │   第三幕  │
├───────────────────────────────────────┤
│ 段落1│段落2│段落3│段落4│段落5│...│    │
├───────────────────────────────────────┤
│ 场景 场景 场景 场景 场景 场景 场景... │
├───────────────────────────────────────┤
│ 节拍 节拍 节拍 节拍 节拍 节拍 节拍... │
└───────────────────────────────────────┘
```

## 🎯 段落的构成

### 典型段落结构
```
┌─────────────────────────────────────────┐
│  起：建立段落目标                       │
│   ↓                                     │
│  承：发展、尝试、障碍                   │
│   ↓                                     │
│  转：转折点、新发现                     │
│   ↓                                     │
│  合：段落小高潮、进入下一段落           │
└─────────────────────────────────────────┘
```

### 段落情感曲线
```
情感强度
    │        ╭──╮ ← 段落高潮
    │       ╱    ╲
    │      ╱      ╲
    │     ╱        ╲
    │────╱          ╲────
    └─────────────────────→ 时间
      起    承    转   合
```

## 🔧 段落类型

### 按功能分类

| 类型 | 功能 | 示例 |
|------|------|------|
| **设置段落** | 建立世界/人物 | 日常生活展示 |
| **发展段落** | 推进情节 | 调查、旅程 |
| **冲突段落** | 对抗障碍 | 对决、争吵 |
| **揭示段落** | 揭露信息 | 真相大白 |
| **转折段落** | 改变方向 | 重大决定 |
| **高潮段落** | 全片最高点 | 最终对决 |
| **解决段落** | 收尾 | 结局展示 |

### 段落长度参考
| 类型 | 时长 | 场景数 |
|------|------|--------|
| 短段落 | 3-5分钟 | 2-3个场景 |
| 标准段落 | 5-10分钟 | 3-5个场景 |
| 长段落 | 10-15分钟 | 5-8个场景 |

## 📋 段落设计模板

### 单一段落设计
```
┌─────────────────────────────────────────┐
│ 段落名称：[名称]                        │
│ 段落目标：[这个段落要达成什么]          │
├─────────────────────────────────────────┤
│ 场景1：[场景描述] - 功能：[起]          │
│ 场景2：[场景描述] - 功能：[承]          │
│ 场景3：[场景描述] - 功能：[承]          │
│ 场景4：[场景描述] - 功能：[转]          │
│ 场景5：[场景描述] - 功能：[合]          │
├─────────────────────────────────────────┤
│ 段落高潮：[高潮时刻描述]                │
│ 进入下一段：[转折/钩子]                 │
└─────────────────────────────────────────┘
```

### 示例：追查段落
```
段落名称：追查真凶
段落目标：主角发现关键线索

场景1：图书馆研究 - 功能：起（开始调查）
场景2：采访证人 - 功能：承（获取信息）
场景3：追踪线索 - 功能：承（接近真相）
场景4：遭遇阻碍 - 功能：转（被警告停止）
场景5：发现关键证据 - 功能：合（突破！）

段落高潮：在证据前的震惊表情
进入下一段：决定对质真凶
```

## 🎯 蒙太奇段落

### 什么是蒙太奇
```
压缩时间的快速剪辑序列
通常配合音乐
展示过程而非细节
```

### 蒙太奇类型
| 类型 | 内容 | 示例 |
|------|------|------|
| **训练蒙太奇** | 能力成长 | 《洛奇》 |
| **准备蒙太奇** | 筹备过程 | 装备、计划 |
| **时间蒙太奇** | 时间流逝 | 四季变化 |
| **关系蒙太奇** | 感情发展 | 约会片段 |
| **对比蒙太奇** | 平行对比 | 贫富对比 |

### 蒙太奇设计模板
```
蒙太奇：[名称]
目的：[压缩展示什么]
配乐：[音乐风格]
时长：[预估]

| 镜号 | 内容 | 时长 | 备注 |
|------|------|------|------|
| 01 | [描述] | 2秒 | 起点 |
| 02 | [描述] | 1秒 | |
| 03 | [描述] | 1.5秒 | |
| ... | ... | ... | |
| 末 | [描述] | 2秒 | 终点 |
```

## 🎬 AI番剧应用

### 段落规划表
```
# [作品名] 段落设计

## 第一幕
### 段落1：日常 (3分钟)
- 场景1.1：早晨起床
- 场景1.2：上学路上
- 场景1.3：教室日常
- 高潮：转学生出现

### 段落2：相遇 (4分钟)
- 场景2.1：被安排同桌
- 场景2.2：初次对话
- 场景2.3：放学同行
- 高潮：发现共同秘密

## 第二幕
### 段落3：...
```

```
训练蒙太奇片段：

片段1："character doing pushups, struggling, sweat"
片段2："character practicing sword, clumsy"
片段3："character reading, late night, tired"
片段4："character doing pushups, stronger"
片段5："character with sword, confident pose"

→ 展示成长过程
```

## 💡 段落设计检查

### 必要性检查
- [ ] 这个段落推进了剧情吗？
- [ ] 删掉会影响理解吗？
- [ ] 有自己的小高潮吗？
- [ ] 连接上下段落吗？

### 节奏检查
- [ ] 段落间节奏有变化吗？
- [ ] 不会连续太多同类型段落？
- [ ] 情感张力有起伏吗？
