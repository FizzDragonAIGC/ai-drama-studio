# 🎤 診斷式提問技能

## 元信息
- **技能ID**: interview_diagnostic
- **所屬Agent**: 訪談Agent
- **參考書籍**: 《提问的力量》Frank Sesno
- **難度**: ⭐⭐⭐

## 核心理念

診斷式提問 = 醫生問診思維
- 不是問「你想要什麼」
- 而是問「哪裡不對勁」

## 8種診斷問題類型

### 1. 症狀問題
```
「這個項目目前最讓你頭疼的是什麼？」
「如果明天必須交付，你最擔心哪個部分？」
```

### 2. 歷史問題
```
「之前有嘗試過類似的項目嗎？結果如何？」
「是什麼觸發了這個創作念頭？」
```

### 3. 對比問題
```
「和市面上的XX相比，你希望有什麼不同？」
「你最喜歡/最討厭的同類作品是？」
```

### 4. 假設問題
```
「如果預算無限，你會怎麼做？」
「如果只能保留一個元素，你選什麼？」
```

### 5. 量化問題
```
「1-10分，你對目前的方向有多確定？」
「預期觀眾看完後，記住幾個關鍵畫面？」
```

### 6. 反向問題
```
「你絕對不想讓這部作品變成什麼樣？」
「觀眾如果這樣理解，你會很失望？」
```

### 7. 優先級問題
```
「視覺風格、故事深度、節奏感，排個序？」
「藝術表達還是商業成功，哪個優先？」
```

### 8. 驗證問題
```
「我理解你是想要...對嗎？」
「所以核心衝突是...這樣總結準確嗎？」
```

## 訪談流程模板

```
第1輪：開放式探索（5分鐘）
├── 「聊聊這個項目的起源？」
├── 「最初的靈感是什麼？」
└── 「為什麼是現在做？」

第2輪：診斷式深挖（10分鐘）
├── 症狀問題 × 2
├── 歷史問題 × 1
├── 對比問題 × 2
└── 反向問題 × 1

第3輪：確認式收斂（5分鐘）
├── 優先級問題 × 1
├── 量化問題 × 2
└── 驗證問題 × 2
```

## 追問技巧

### 5Why深挖法
```
用戶：「我想要電影感」
追問1：「為什麼電影感很重要？」
→ 「因為想讓觀眾沉浸」
追問2：「為什麼沉浸感是關鍵？」
→ 「因為故事有很多細節」
追問3：「為什麼這些細節必須被注意到？」
→ 「因為結局的反轉需要前面的伏筆」
= 真正需求：伏筆-反轉結構，需要引導觀眾注意力
```

### 具象化追問
```
用戶：「我想要那種感覺」
追問：「能舉個具體的電影/動畫片段嗎？」
追問：「那個片段裡，哪個鏡頭最打動你？」
追問：「如果要用一個形容詞，你會說？」
```

## 常見陷阱

| 陷阱 | 正確做法 |
|------|----------|
| 用戶說「都行」 | 給出二選一強迫選擇 |
| 用戶說「像XX一樣」 | 追問「像XX的哪個方面」 |
| 用戶自相矛盾 | 指出矛盾，讓其排序 |
| 用戶不確定 | 用假設問題打開思路 |
