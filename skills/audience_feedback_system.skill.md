# 🔄 观众反馈系统 Skill
> 模拟观众视角的质量评审系统

---

## 📐 反馈系统架构

### 评审流程
```
创作完成
    ↓
┌─────────────────────────────────────┐
│         观众模拟器评审               │
├─────────────────────────────────────┤
│  1. 目标观众是谁？                   │
│  2. 他们看到会有什么反应？           │
│  3. 评分（1-10）                    │
│  4. 问题在哪？                       │
│  5. 如何改进？                       │
└─────────────────────────────────────┘
    ↓
修改 ──→ 再评审 ──→ 通过
```

## 🎯 评审维度

### 剧本评审
| 维度 | 评分标准 | 权重 |
|------|----------|------|
| 钩子力 | 前30秒能否抓住人 | 20% |
| 节奏感 | 是否拖沓/太赶 | 15% |
| 情感共鸣 | 能否引发情绪 | 20% |
| 角色魅力 | 角色是否讨喜 | 15% |
| 悬念设计 | 想不想看下一集 | 15% |
| 逻辑性 | 剧情是否合理 | 10% |
| 独特性 | 有没有新意 | 5% |

### 视觉评审
| 维度 | 评分标准 | 权重 |
|------|----------|------|
| 第一印象 | 画面是否吸引人 | 25% |
| 角色颜值 | 角色是否好看 | 20% |
| 场景氛围 | 氛围是否到位 | 20% |
| 镜头语言 | 是否专业 | 15% |
| 色彩情绪 | 色彩是否配合情绪 | 10% |
| 细节品质 | 细节是否精致 | 10% |

## 📊 观众模拟器提问

### 通用问题
```
1. 目标观众是谁？
   → [年龄/性别/偏好描述]

2. 这个观众看到第一眼会想什么？
   → [模拟内心独白]

3. 他会继续看下去吗？为什么？
   → [分析原因]

4. 看到这里他的情绪是？
   → [情绪+强度]

5. 他会给几分？（1-10）
   → [评分+理由]

6. 他会吐槽什么？
   → [预测差评内容]

7. 如何改进能让他更满意？
   → [具体建议]
```

### 分环节问题

**剧本环节：**
```
- 故事有吸引力吗？
- 角色讨喜吗？
- 节奏合适吗？
- 对白自然吗？
- 想看下一集吗？
```

**角色设计环节：**
```
- 角色好看吗？
- 角色有特点吗？
- 符合人设吗？
- 会想cos吗？
- 会想画同人吗？
```

**场景设计环节：**
```
- 场景有氛围吗？
- 场景符合剧情吗？
- 场景有记忆点吗？
- 会想截图吗？
```

**分镜环节：**
```
- 镜头有张力吗？
- 情绪表达到位吗？
- 视觉有冲击力吗？
- 节奏感好吗？
```

## 📋 迭代优化流程

### 单次迭代
```
第1版 → 评审 → 发现问题 → 修改 → 第2版 → 再评审
```

### 多维度迭代
```
剧本评审 → 剧本修改 → 通过
    ↓
角色评审 → 角色修改 → 通过
    ↓
场景评审 → 场景修改 → 通过
    ↓
分镜评审 → 分镜修改 → 通过
    ↓
整体评审 → 微调 → 完成
```

## 🔧 使用方法

### 在Prompt Generator中集成
```
每个Agent完成工作后，自动触发：
1. 观众画像师确认目标观众
2. 情感分析师预测情绪反应
3. 质量评审员打分
4. 如果<8分，给出修改建议
5. 修改后再次评审
6. 通过后进入下一环节
```

### 评审Prompt模板
```
请以[目标观众]的视角，评审以下内容：

[内容]

请回答：
1. 评分（1-10）：

3. 优点（3条）：
4. 问题（3条）：
5. 改进建议（3条）：
6. 预测弹幕：
7. 是否通过：
```

## 📊 质量标准

| 评分 | 含义 | 行动 |
|------|------|------|
| 9-10 | 优秀 | 直接通过 |
| 7-8 | 良好 | 小修后通过 |
| 5-6 | 及格 | 需要修改 |
| 3-4 | 较差 | 需要重做部分 |
| 1-2 | 很差 | 需要完全重做 |
