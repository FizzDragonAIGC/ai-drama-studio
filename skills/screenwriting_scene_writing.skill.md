# 场景写作 Skill
> 来源：《故事》Robert McKee、《救猫咪》系列
> 适用智能体：编剧助手、分镜设计师

## 核心理念

**"每个场景都是一个迷你故事"** —— 有开头、发展、结尾，有变化

场景（Scene）是剧本的基本单位，好的场景必须推动故事前进。

---

## 核心理论

### 场景的定义

```
场景 = 同一时间 + 同一地点 + 连续动作
时空任一改变 = 新场景
```

### 场景的本质：价值转变

**核心公式**：
```
场景开始的价值状态 ≠ 场景结束的价值状态
```

**价值维度示例**：
- 爱 ↔ 恨
- 希望 ↔ 绝望
- 信任 ↔ 背叛
- 安全 ↔ 危险
- 真相 ↔ 谎言

### 场景功能清单

每个场景至少完成一项：
1. ✅ 推动情节
2. ✅ 揭示角色
3. ✅ 提供信息
4. ✅ 建立氛围
5. ✅ 表达主题

## 实操技巧

### 场景写作三步法

**Step 1：明确进场**
- 角色带着什么目标进入场景？
- 场景开始时价值状态是什么？

**Step 2：设计冲突**
- 什么阻碍角色实现目标？
- 冲突升级的节奏如何？

**Step 3：确定出场**
- 场景如何改变价值状态？
- 角色带着什么离开场景？

### 场景结构

```
┌─────────────────────────────────────┐
│ 1. 建立（Setup）                    │
│    - 时空交代                       │
│    - 人物位置                       │
│    - 开场状态                       │
├─────────────────────────────────────┤
│ 2. 发展（Development）              │
│    - 目标展现                       │
│    - 障碍出现                       │
│    - 冲突升级                       │
├─────────────────────────────────────┤
│ 3. 转折/高潮（Turning Point）       │
│    - 价值转变                       │
│    - 新信息/新状态                  │
└─────────────────────────────────────┘
```

### 场景节奏控制

| 场景类型 | 节奏 | 技巧 |
|----------|------|------|
| 动作场景 | 快 | 短句、快切、少对话 |
| 情感场景 | 慢 | 留白、反应、沉默 |
| 悬疑场景 | 张弛 | 紧张-缓和-更紧张 |
| 喜剧场景 | 明快 | 节奏准确、包袱密度 |

### 场景转换技巧

| 转换类型 | 说明 | 示例 |
|----------|------|------|
| 硬切 | 直接跳转 | 战斗→教室 |
| 匹配剪辑 | 视觉/动作匹配 | 挥剑→切蛋糕 |
| 声音转场 | 声音先行/延续 | 爆炸声→闹钟声 |
| 情绪转场 | 情绪状态连接 | 绝望→黎明 |
| 因果转场 | 上场结果引出下场 | 发现线索→追踪 |

## Prompt模板

### 场景生成

```
请为以下情境写一个完整场景：

场景设定：
- 时间：[时间]
- 地点：[地点]
- 人物：[在场角色]
- 开场价值状态：[如：主角信任配角]
- 结束价值状态：[如：主角开始怀疑配角]

要求：
1. 包含建立-发展-转折三段
2. 通过行动和对话展示，不要直接说明
3. 有明确的价值转变
4. 长度约500字
5. 番剧风格，适合视觉化
```

### 场景诊断

```
请诊断以下场景的问题：

[粘贴你的场景]

分析维度：
1. 场景是否有明确的价值转变？
2. 冲突是否足够？
3. 是否有不必要的内容可以删除？
4. 进场/出场是否有力？
5. 节奏是否合适？
6. 修改建议
```

### 场景序列设计

```
为以下情节点设计场景序列：

情节目标：[需要达成的叙事目标]
涉及角色：[角色列表]
当前状态：[故事当前状态]

请设计3-5个场景的序列：
每个场景包含：
- 时空设定
- 主要人物
- 核心动作
- 价值转变
- 与下一场景的连接
```

## 检查清单

### 单场景检查
- [ ] 有明确的价值转变？
- [ ] 进场目标清晰？
- [ ] 有实质性冲突？
- [ ] 出场状态改变？
- [ ] 可视觉化呈现？

- [ ] 紧张度有起伏？
- [ ] 没有冗余场景？
- [ ] 时空转换流畅？

## 案例分析

### 案例：《咒术回战》渋谷站战斗场景

**场景设定**：
- 时间：夜晚
- 地点：涩谷地铁站
- 人物：虎杖、真人

**价值转变**：
- 开始：虎杖愤怒但有控制
- 结束：虎杖彻底失控，黑闪连发

**场景结构分析**：

| 阶段 | 内容 | 技巧 |
|------|------|------|
| 建立 | 真人嘲讽，虎杖压抑愤怒 | 慢节奏，积蓄情绪 |
| 发展 | 回忆穿插，真人攻击 | 闪回对比 |
| 高潮 | 虎杖黑闪爆发 | 动态加速，特效爆发 |

**经验提炼**：
1. 情绪场景需要酝酿时间
2. 闪回可以加深情感重量
3. 高潮释放要与铺垫匹配

### 场景写作示例

```
INT. 屋顶 - 黄昏

【建立】
夕阳染红天际。小明独自站在护栏边，手机紧握。

【发展】
小红推门上来。
        小红
    你还在躲什么？
小明不回头。
        小明
    你不该来。
        小红
    你打算一辈子不说吗？
小明转身，眼眶泛红。

【转折】
        小明
    说了又怎样？你已经订婚了。
小红愣住。手机从小明手中滑落。
屏幕上是小红的订婚照片。

价值转变：隐瞒 → 真相揭露
```

## 常见错误

### ❌ 静态场景
- 人物聊了半天，什么都没改变
- **解决**：确保有价值转变

### ❌ 信息堆砌
- 把该说的信息都堆在一个场景
- **解决**：分散到多个场景，通过行动展示

### ❌ 空洞冲突
- 为了冲突而冲突，与主线无关
- **解决**：每个冲突都应推动角色/情节

### ❌ 拖沓进场
- 花太多时间"进入"场景
- **解决**：尽量晚进场、早离场
