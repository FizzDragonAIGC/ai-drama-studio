# ✍️ 旁白設計技能

## 元信息
- **技能ID**: script_voiceover
- **所屬Agent**: 劇本Agent
- **參考書籍**: 《故事》+ 《電影劇本寫作》
- **難度**: ⭐⭐⭐

## 核心理念

旁白 = 需要謹慎使用的工具
- 能用畫面說的，不用旁白
- 旁白是「加分項」不是「救急」

## 旁白類型

### 1. 角色旁白（第一人稱）
```
主角的內心聲音
例：《肖申克的救贖》

「我至今還記得那個下雨的夜晚...」

效果：親密感、主觀視角
```

### 2. 敘述者旁白（第三人稱）
```
全知視角的講述
例：《天使艾米麗》

「艾米麗從小就與眾不同...」

效果：客觀感、童話感
```

### 3. 書信/日記旁白
```
角色朗讀書面內容
例：《情書》

「你好嗎？我很好。」

效果：跨越時空的連接
```

### 4. 解說旁白
```
介紹背景/規則
例：紀錄片風格

「這是湘西小鎮，一百年來...」

效果：建立世界觀
```

## 旁白使用時機

### ✅ 適合使用旁白
```
- 快速建立時代/背景
- 大量時間跳躍
- 角色內心無法外化時
- 風格需要（文學改編）
- 製造反諷效果
```

### ❌ 避免使用旁白
```
- 能用對話/行動表達時
- 解釋剛才發生的事
- 彌補劇本的不足
- 過度使用（觀眾會累）
```

## 旁白寫作技巧

### 1. 簡潔
```
❌ 「那是一個風和日麗的下午，陽光明媚，
    我走在回家的路上，心情非常愉快...」

✅ 「那個下午，一切都將改變。」
```

### 2. 與畫面對位
```
對照：旁白說A，畫面呈現B（反諷）
「我很幸福。」+ 畫面：她在哭

同步：旁白補充畫面
「三年後」+ 畫面：老了的角色
```

### 3. 有節奏
```
旁白不是連續不斷的
有進入、有停頓、有退出
```

### 4. 有個性
```
角色旁白要有角色的語氣
不是作者在說，是角色在說
```

## 旁白格式

### 劇本格式
```
小明（V.O.）
那年我十七歲，還不懂什麼是離別。

V.O. = Voice Over（畫外音）
O.S. = Off Screen（畫面外，但在場景中）
```

### AI番劇格式
```json
{
  "voiceover": {
    "speaker": "翠翠",
    "type": "角色旁白",
    "text": "爺爺說，命是天定的。但我總覺得，命是等來的。",
    "timing": "場景開始",
    "tone": "平靜、略帶憂傷",
    "sync_with": "翠翠望向渡口"
  }
}
```

## 旁白與畫面的關係

| 關係 | 效果 | 例子 |
|------|------|------|
| 同步 | 強化 | 說「雨」，畫面是雨 |
| 補充 | 延伸 | 說背景，畫面是行動 |
| 對照 | 反諷 | 說「開心」，畫面是悲傷 |
| 預示 | 懸念 | 說未來，畫面是現在 |
