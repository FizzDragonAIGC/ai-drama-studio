# Robert McKee 序列式章節結構
> 基於 Robert McKee《Story》的 Beat-Scene-Sequence-Act 結構

## 📚 理論來源
Robert McKee 被譽為「編劇教父」，他的《Story》是電影學院必讀經典。

## 🎯 核心理念
**故事是由一系列價值變化組成的。每個場景都必須改變某個價值。**

---

## 📋 McKee 的故事層級結構

```
故事 (Story)
  ├── 幕 (Act) - 3-5個
  │     ├── 序列 (Sequence) - 每幕2-5個
  │     │     ├── 場景 (Scene) - 每序列2-5個
  │     │     │     └── 節拍 (Beat) - 每場景多個
```

### 層級定義

| 層級 | 定義 | 價值變化程度 |
|------|------|-------------|
| **Beat（節拍）** | 行動/反應的最小單位 | 微小變化 |
| **Scene（場景）** | 在統一時空中的衝突 | 小型價值轉變 |
| **Sequence（序列）** | 一系列場景組成的迷你故事 | 中型價值轉變 |
| **Act（幕）** | 多個序列的高潮 | 重大價值轉變 |
| **Story（故事）** | 整個作品 | 終極價值轉變 |

---

## 🎬 章節拆分實戰

### 每章 = 一個 Sequence（序列）

McKee認為，一個序列是一個「迷你電影」，有自己的：
- 開頭（設置衝突）
- 中段（升級衝突）
- 結尾（解決或轉折）

### 序列設計原則

1. **統一目標** - 每個序列圍繞一個清晰目標
2. **價值翻轉** - 開頭和結尾的價值狀態相反
3. **因果鏈** - 每個場景因果相連
4. **情緒統一** - 序列有統一的情緒基調

### 價值變化範例

| 序列 | 開頭價值 | 結尾價值 | 變化 |
|------|----------|----------|------|
| 追求 | 孤獨(-) | 希望(+) | 負→正 |
| 背叛 | 信任(+) | 懷疑(-) | 正→負 |
| 覺醒 | 無知(-) | 認知(+) | 負→正 |
| 墮落 | 正義(+) | 腐敗(-) | 正→負 |

## 📊 場景檢查法

McKee的場景分析三步驟：

### 1. 列出節拍 (Beat Outline)
將場景拆解為一系列行動/反應：
- 他說... → 她反應...
- 她做... → 他反擊...

### 2. 找轉折點 (Turning Point)
場景中價值變化最大的那一刻：
- 「我知道是你幹的」（揭露）
- 「我不會原諒你」（決裂）

### 3. 驗證必要性
問自己：
- 刪掉這場戲，故事會垮嗎？
- 觀眾會發現少了什麼嗎？
- 如果不會，這場戲就該刪。
