# 電影劇本寫作 - Film Screenplay Writing

## 核心書籍方法論

### 1. Syd Field《Screenplay》三幕結構
**電影劇本的奠基之作**

```
第一幕 (Setup) - 25%
├── 開場 (1-10頁): 建立世界、引入主角
├── 觸發事件 (12-15頁): 打破日常的事件
└── 第一轉折點 (25-30頁): 主角被迫行動

第二幕 (Confrontation) - 50%
├── 上升動作: 主角追求目標
├── 中點 (60頁): 重大轉折，提高賭注
├── 下降動作: 挫折與考驗
└── 第二轉折點 (85-90頁): 最黑暗時刻

第三幕 (Resolution) - 25%
├── 高潮: 最終對決
└── 結局: 新的平衡
```

### 2. Robert McKee《Story》序列結構
**場景→序列→幕的層級結構**

```
Beat (節拍) → Scene (場景) → Sequence (序列) → Act (幕)

序列 = 2-5個場景，完成一個小目標
每幕 = 3-5個序列
```

**價值轉換公式：**
```
每個場景必須有價值轉換
開始狀態 → 衝突 → 結束狀態（必須不同）
例：希望→挫折、安全→危險、愛→恨
```

### 3. Blake Snyder《Save the Cat》15節拍
**商業電影的黃金公式**

| 節拍 | 頁數 | 內容 |
|------|------|------|
| 1. Opening Image | 1 | 主題視覺隱喻 |
| 2. Theme Stated | 5 | 主題台詞 |
| 3. Set-up | 1-10 | 建立世界 |
| 4. Catalyst | 12 | 觸發事件 |
| 5. Debate | 12-25 | 主角猶豫 |
| 6. Break into Two | 25 | 進入新世界 |
| 7. B Story | 30 | 情感副線 |
| 8. Fun and Games | 30-55 | 類型承諾 |
| 9. Midpoint | 55 | 假勝利/假失敗 |
| 10. Bad Guys Close In | 55-75 | 危機加劇 |
| 11. All Is Lost | 75 | 最低點 |
| 12. Dark Night of Soul | 75-85 | 內心煎熬 |
| 13. Break into Three | 85 | 頓悟 |
| 14. Finale | 85-110 | 高潮 |
| 15. Final Image | 110 | 對應開場 |

## 電影劇本格式規範

### 場景標題 (Scene Heading)
```
INT. 咖啡廳 - 日

EXT. 大街 - 夜

INT./EXT. 汽車（移動中）- 黃昏
```

### 動作描寫 (Action)
```
- 現在時態
- 簡潔有力，一段不超過4行
- 只寫能「看到」和「聽到」的
- 避免形容詞堆砌
```

### 對白格式 (Dialogue)
```
                    角色名
            （括號語/語氣）
        對白內容，置中對齊
```

### 頁數與時長
```
1頁 ≈ 1分鐘
90-120頁 = 90-120分鐘
商業片：100-110頁最佳
```

## 電影類型結構

### 動作片
- 每10分鐘一個動作場景
- 開場5分鐘內必有動作
- 高潮動作持續15-20分鐘

### 愛情片
- 相遇 → 相知 → 阻礙 → 分離 → 重逢
- 中點通常是「定情」或「誤會」

### 驚悚/懸疑
- 開場即懸念
- 每幕結尾必有twist
- 真相揭露在倒數第二場

### 喜劇
- 每3頁一個笑點
- 誤會是核心引擎
- 高潮是最尷尬的場面

## AI劇本生成Prompt模板

```
請為以下故事生成電影劇本：

【類型】：動作/愛情/驚悚/喜劇
【時長】：90/100/120分鐘
【主題】：一句話主題
【Logline】：一句話故事

要求：
1. 嚴格按照三幕結構
2. 第一幕25%，第二幕50%，第三幕25%
3. 包含15個Save the Cat節拍
4. 每場景有明確的價值轉換
5. 輸出標準劇本格式
```
