# ✍️ 轉場設計技能

## 元信息
- **技能ID**: script_transition
- **所屬Agent**: 劇本Agent
- **參考書籍**: 《剪輯的語法》Roy Thompson
- **難度**: ⭐⭐⭐

## 核心理念

轉場 = 場景之間的橋樑
- 不只是技術處理
- 是敘事節奏的一部分

## 轉場類型

### 1. 硬切（Cut）
```
最常用：場景A直接切到場景B
效果：乾淨、直接
適合：連續動作、對比、快節奏

劇本格式：（不需要標註，默認就是硬切）
```

### 2. 疊化（Dissolve）
```
場景A漸漸消失，場景B漸漸出現
效果：時間流逝、夢幻、回憶
適合：時間跳躍、情緒過渡

劇本格式：
疊化至：

或：

...畫面疊化...
```

### 3. 淡入/淡出（Fade）
```
淡出黑/白 → 淡入新場景
效果：段落結束、重大時間跳躍
適合：章節分隔、結尾

劇本格式：
淡出。
淡入：
```

### 4. 匹配切換（Match Cut）
```
場景A的元素 = 場景B的元素
效果：視覺連續、暗示關聯

例：
A：旋轉的輪子
B：旋轉的地球
= 從微觀到宏觀的關聯

劇本格式：
匹配切換至：
```

### 5. 聲音橋接（Sound Bridge）
```
場景B的聲音在場景A結尾出現
或：場景A的聲音延續到場景B

效果：流暢、預示

例：
A：她掛掉電話
B的聲音：（電話那頭的聲音開始）
切到B：他拿著電話
```

## 創意轉場技術

### 視覺相似
```
圓形物體 → 另一個圓形
（眼睛 → 月亮）
```

### 動作連續
```
A：他揮拳
B：另一個人完成揮拳
```

### 顏色過渡
```
A：紅色的夕陽
B：紅色的信封
```

### 遮擋轉場
```
A：角色走過鏡頭（黑屏）
B：新場景從黑屏展開
```

### 概念連接
```
A：他說「我要去找她」
B：他已經站在她門前
```

## 轉場與節奏

### 快節奏段落
```
- 硬切為主
- 場景短促
- 少用過渡效果
```

### 慢節奏段落
```
- 疊化、淡化
- 場景較長
- 允許「喘息」
```

### 情緒轉換
```
喜劇→悲劇：硬切（對比衝擊）
悲劇→希望：疊化（慢慢過渡）
```

## AI番劇轉場標註

```json
{
  "transition": {
    "from_scene": 5,
    "to_scene": 6,
    "type": "匹配切換",
    "description": "翠翠眼中的淚水 → 河面的倒影",
    "duration": "1秒",
    "audio_treatment": "環境音延續"
  }
}
```

## 轉場禁忌

```
❌ 過度使用花哨轉場
❌ 轉場打斷敘事節奏
❌ 沒有邏輯的隨意轉場
❌ 每個場景都用疊化

✅ 硬切是默認選擇
✅ 特殊轉場有明確目的
✅ 轉場服務於敘事
```
