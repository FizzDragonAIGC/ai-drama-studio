# 🔄 观众反馈闭环系统 v1.0

> 每个创作环节完成后，都要经过"观众模拟器"的评审
> 不是我们觉得好，是观众觉得好

## 各环节评审标准

### 1. 剧本评审
| 维度 | 问题 | 权重 |
|------|------|------|
| 钩子力 | 前30秒能抓住人吗？ | 20% |
| 情感共鸣 | 观众会有感觉吗？ | 20% |
| 节奏感 | 是否拖沓？是否太赶？ | 15% |
| 角色魅力 | 主角讨喜吗？ | 15% |
| 悬念感 | 想看下一集吗？ | 15% |
| 逻辑性 | 剧情合理吗？ | 10% |
| 独特性 | 有什么新意？ | 5% |

**评审输出：**
```
剧本评审报告
━━━━━━━━━━━━━━━
目标观众：[画像描述]
情绪预测：[情绪曲线]
总分：8.5/10

观众预期反应：
"这个开头不错，想继续看..."

优点：
1. 钩子强，前30秒有悬念
2. 主角设定讨喜
3. 有反转设计

问题：
1. 中间节奏有点慢（严重度：中）
2. 配角动机不清晰（严重度：低）

修改建议：
1. 第3分钟-第5分钟压缩
2. 增加配角背景铺垫

预测弹幕：
"开头可以！" "主角好惨" "快反杀！"

结论：【通过/小修】
```

```

- 年龄：[年龄]
- 性别：[性别]
- 偏好：[偏好描述]
- 观看场景：[在哪看、什么时候看]

【内容】
[待评审内容]

1. 你的第一反应是什么？（一句话）
2. 总体评分（1-10分）：
3. 你喜欢什么？（3点）
4. 你不喜欢什么？（3点）
5. 如果这样改，你会更喜欢：（3条建议）

[内容]

评分：X/10

主要问题：[最大的问题]

```

## 评分标准

| 评分 | 含义 | 行动 |
|------|------|------|
| 9-10 | 🌟 优秀 | 直接通过，考虑作为标杆 |
| 8-8.9 | ✅ 良好 | 通过 |
| 7-7.9 | ⚠️ 及格 | 小修后通过 |
| 5-6.9 | ❌ 不足 | 需要修改后重评 |
| <5 | ❌❌ 较差 | 需要重做 |

## 实施指南

### 在Prompt Generator中集成
每个Agent输出后，自动调用观众评审Agent：

```javascript
// 伪代码
async function processStep(agentOutput) {
  // 1. 获取观众画像
  const persona = await A26_getAudiencePersona(projectContext);
  
  // 2. 情感预测
  const emotionPrediction = await A27_predictEmotion(agentOutput, persona);
  
  // 3. 质量评审
  const review = await A30_qualityReview(agentOutput, persona);
  
  // 4. 判断是否通过
  if (review.score >= 8) {
    return { status: 'pass', output: agentOutput };
  } else {
    // 5. 生成修改建议
    const suggestions = review.suggestions;
    return { status: 'revise', suggestions };
  }
}
```

### 手动使用
每个环节完成后，用以下流程：

1. **定义观众**：这个内容给谁看？
2. **模拟反应**：他们看到会怎么想？
3. **打分**：1-10分
4. **找问题**：哪里不够好？
5. **给建议**：怎么改更好？
6. **迭代**：修改后再评
